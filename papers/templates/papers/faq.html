{% trans "Open Access" %}

{% trans "What is Open Access?" %}

{% blocktrans trimmed %} In a nutshell, the Open Access movement promotes free access to the scientific literature. For a more in-depth introduction, we recommend this video in English and this introduction in French. {% endblocktrans %}

{% trans "Why should I upload my papers to a repository?" %}

{% blocktrans trimmed %} You should do it because:

  • It makes your research more accessible and more visible. Many researchers routinely check papers in topic repositories such as arXiv, and it is less likely that these researchers check your homepage regularly.
  • You can even do it before publication. You can get valuable feedback to improve your paper before the actual peer-reviewing process. It also witnesses the fact that you have had your ideas earlier than the actual publication time.
  • It provides you a durable storage. Your homepage is likely to change or to disappear, for instance if you leave your current institution.
  • Some publishers do not allow preprints on homepages but allow them in repositories.
  • Finally, for all these reasons, an increasing number of funding bodies require the outcomes of their projects to be available in a repository.
{% endblocktrans %}

{% trans "What repository should I use?" %}

{% blocktrans trimmed %} There are roughly two types of repositories:

  • Subject repositories, restricted to some research fields, such as arXiv.
  • Institutional repositories, designated to store the research output of a given institution or country. For instance, {{ UNIVERSITY_SHORT_NAME }} has its own repository.
If your research community favours a specific repository, using it makes sense as it will increase your visibility. If not, institutional repositories are also a good option. {% endblocktrans %}

dissem.in

{% trans "What is dissem.in?" %}

{% url 'sources' as sources_url %} {% blocktrans trimmed %} Dissem.in is a web platform gathering metadata from various sources and analyzing the full text availability of publications of researchers, currently only within {{ UNIVERSITY_SHORT_NAME }}. It has been designed to foster the use of repositories (rather than preprints posted on personal homepages), for numerous reasons. {% endblocktrans %}

{% trans "Is dissem.in a repository?" %}

{% blocktrans trimmed %} Dissem.in is not a repository as it does not store any full text. When the full text of a publication is available, a link to the relevant page is provided. {% endblocktrans %}

{% trans "Who runs dissem.in?" %}

{% blocktrans trimmed %} Dissem.in is brought to you by the dissem.in team. {% endblocktrans %}

{% trans "Can I create my profile on dissem.in?" %}

{% blocktrans trimmed %} If your institution is not covered by dissem.in, currently, no, you cannot. Our algorithms require some information about your department and research field to gather your publications. {% endblocktrans %}

{% trans "Can I use dissem.in for my institution?" %}

{% blocktrans trimmed %} Yes, just ask contact@dissem.in. {% endblocktrans %}

Policies and full-text availability

{% trans "What do your policy classes mean?" %}

{% blocktrans trimmed %} Open access:
The full text is made freely available by the publisher. {% endblocktrans %}
{% blocktrans trimmed %} Pre/post prints allowed and full text available:
The full text is available in a repository, but might not be freely available from the publisher itself. This is in line with the publishing policy and is called green open access or self-archiving. {% endblocktrans %}
{% blocktrans trimmed %} Pre/post prints allowed but full text unavailable:
The full text could be made available in a repository (as above) according to the publishing policy. {% endblocktrans %}
{% blocktrans trimmed %} Pre/post prints forbidden:
The publisher forbids deposits in repositories, in general. {% endblocktrans %}

{% trans "When is a paper considered available?" %}

{% blocktrans trimmed %} When we have found it in one of the repositories we harvest. The list of these repositories can be found here. Thanks to the BASE search engine, we cover more than 68 millions of preprints. {% endblocktrans %}

{% trans "Should I trust your statistics?" %}

{% blocktrans trimmed %} There are a few caveats concerning our data:

  • Some papers are not present in any of our metadata sources, so they are not counted at all.
  • Some entries in our system do not correspond to proper research papers.
  • Some of them have been incorrectly classified by our homonym disambiguation system.
  • Some repositories might not be covered by any of our sources, so a paper might be marked as unavailable while it is available.
  • Some papers might be marked as available but actually, they are not (for instance because the content has been deleted).
  • And many other technical issues.
But our platform is increasingly reliable and we think that department-wise statistics are meaningful. {% endblocktrans %}

Omissions and errors

{% trans "One of my papers is missing. Why?" %}

{% blocktrans trimmed %}

There are many possible reasons: maybe it is not available from any of our sources, your name is spelt differently or partially in the authors list, our algorithms estimated that it has been written by an homonym researcher, or it has been deleted by one of your co-authors.

In any case, you should be able to add it manually soon.

{% endblocktrans %}

{% trans "Some details of my paper are incorrect. What can I do?" %}

{% blocktrans trimmed %} In a near future, you should be able to create an account and fix it by yourself. If it is really urgent, send a message to contact@dissem.in. {% endblocktrans %}

{% trans "There is a freely available copy of my article online but dissem.in does not find it." %}

{% blocktrans trimmed %} There are two cases:

  • Your article is available on a personal homepage or a similar website. In this case, you should read why you should also put it in a repository.
  • Your article is available in a repository but does not appear as available in dissem.in. In this case you should send a message to contact@dissem.in to fix the problem.
{% endblocktrans %}
